Getting Started

You can get started with using flexCarousel.js by downloading the required files from the GitHub repository.

The only dependency that flexCarousel.js has is jQuery which you need to load before the plugin. After you have loaded both jQuery and flexCarousel.js, you need to initialize the plugin.

Compabitility

flexCarousel.js uses a modern browser technology called Flexbox. According to Can I use, this jQuery plugin is compatible with recent versions of the following browsers:

  • Chrome
  • Edge
  • Firefox
  • Opera
  • Safari

Internet Explorer is only partially supported.

Options

  • arrows: (default true). Determines whether a previous and next arrow should be shown on either side of the slides so that the user can change the active slide.
  • arrowsOverlay: (default true). arrows must be set to true for this option to work. Determines whether the arrows overlay onto the active slide or have their own wrapping elements.
  • autoplay: (default false). Determines whether the active slide changes to the next slide on a time delay.
  • autoplaySpeed: (default 5000). autoplay must be set to true for this option to work. Determines how long the active slide will stay active for before transitioning to the next slide.
  • height: (default: null). Defines whether a fixed height is to be added to the .fc parent element.
  • loop: (default: true). Determines whether the carousel should continously scroll or stop at the last slide.
  • nextArrow: (default '<i class="fas fa-angle-right"></i>'). Defines the HTML code for the element which changes the active slide to the next slide when clicked on. Out of the box, flexCarousel.js supports Font Awesome 5.
  • prevArrow: (default '<i class="fas fa-angle-left"></i>'). Defines the HTML code for the element which changes the active slide to the previous slide when clicked on. Out of the box, flexCarousel.js supports Font Awesome 5.
  • Experimental slidesVisible: (default 1). Defines how many slides are visible at any given time.
  • transition: (default 'slide'). Defines which transition should be used when changing the active slide. Current transitions available are: 'none' and 'slide'.
